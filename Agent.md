# BambooFlute Agent Guide

## Project Goal

Build a Unity-based bamboo flute practice app for Android beginners.

Current milestone:

1. Pitch/tuning practice
2. Metronome

## Current Architecture

- Engine: Unity 6
- Runtime audio input: `Microphone`
- Pitch detection: custom C# YIN-style detector
- UI approach: static prefabs generated by an editor builder, not runtime-generated UI
- Main scene: `Assets/Scenes/SampleScene.unity`
- Main prefabs:
  - `Assets/Prefabs/MainUI.prefab`
  - `Assets/Prefabs/TunerPanel.prefab`
  - `Assets/Prefabs/MetronomePanel.prefab`

## Important Scripts

- `Assets/Scripts/Runtime/PracticeNavigationController.cs`
  - Home page navigation between tuner and metronome
- `Assets/Scripts/Runtime/TunerPanelController.cs`
  - Tuner page interaction and display updates
- `Assets/Scripts/Runtime/BambooFluteTargetLibrary.cs`
  - Flute key, tonguing mode, and target-note option generation
- `Assets/Scripts/Runtime/MicrophonePitchTracker.cs`
  - Microphone permission, sampling, and smoothed pitch tracking
- `Assets/Scripts/Runtime/PitchDetector.cs`
  - Frequency detection core
- `Assets/Scripts/Runtime/PitchMath.cs`
  - Note mapping and cents math
- `Assets/Scripts/Runtime/Metronome.cs`
  - Audio click generation and beat scheduling
- `Assets/Scripts/Runtime/MetronomePanelController.cs`
  - Metronome page UI control
- `Assets/Editor/StaticUiBuilder.cs`
  - Rebuilds the static prefabs and refreshes `SampleScene`

## Tuner UX Direction

The tuner is now split into two tabs:

- `听音`
  - Smart listening / auto-detect mode
  - Uses current detected pitch as the comparison basis
- `目标音`
  - Practice mode with explicit target selection
  - Flute key should be direct-select buttons, not a slider
  - `筒音作5 / 筒音作2` is a small toggle
  - Target notes are arranged in `低音 / 中音 / 高音` rows
  - Each note is shown as a tappable circle with text tied to that same node

The gauge should follow a normal half-circle:

- left = low
- top = in tune
- right = high

## UI Generation Workflow

When changing prefab layout, the source of truth is the editor builder:

- Update `Assets/Editor/StaticUiBuilder.cs`
- In Unity run `Tools/BambooFlute/Rebuild Static UI`

That tool should:

1. Rebuild `MainUI.prefab`
2. Rebuild `TunerPanel.prefab`
3. Rebuild `MetronomePanel.prefab`
4. Replace the scene instances under the scene `Canvas`

## Fonts

- Main TMP font: `Assets/Font/SourceHanSansCN-Regular SDF.asset`
- Character list source: `Assets/Font/font.txt`

If new Chinese text is added:

1. update `font.txt`
2. regenerate/update the TMP atlas if glyphs are missing

## Current Known UX Constraints

- The user prefers direct point-and-select interactions over sliders for target-note practice
- Tuner visuals are being iterated toward a reference-app style gauge
- Circle note selectors should have:
  - larger invisible hit area
  - smaller visible dot
  - labels visually attached to the same node for alignment

## Recommended Next Steps

- Keep tuning UI work in `StaticUiBuilder.cs` and `TunerPanelController.cs`
- Keep note option logic in `BambooFluteTargetLibrary.cs`
- If adding new practice modes, avoid breaking the existing `听音 / 目标音` split
- Test all prefab structure changes by rebuilding static UI before judging runtime behavior

## Notes For Future Agents

- Avoid reintroducing runtime-generated full-page UI
- Prefer editing the builder instead of hand-editing prefab YAML
- Preserve the user’s preference for beginner-friendly, highly direct controls
- When touching pitch logic, verify silence handling, noisy attacks, and tuner stability
